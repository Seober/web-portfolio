# 개인 포트폴리오 웹사이트 기획서

## Context (왜 만드는가)

C++ 게임 개발자인 사용자의 커리어를 한 눈에 보여주는 개인 포트폴리오 웹사이트. 채용 담당자나 동료에게 링크 하나로 이력·자기소개·프로젝트 이력 전체를 전달하는 것이 목적. 기존 포트폴리오 사이트들은 1) 필요 이상으로 화려하거나 2) 단순히 텍스트 나열에 그쳐서, **"미니멀하지만 시각적 타임라인을 중심으로 커리어 흐름을 보여주는"** 사이트가 필요함. 추가로, 새로운 프로젝트가 완료될 때마다 마크다운 파일만 추가하면 자동으로 타임라인에 반영되도록 하고, 타임라인 상단에는 **"Coming soon" 플레이스홀더 카드**를 두어 "지속적으로 업데이트되고 있다"는 인상을 줌. 사용자가 웹 프론트엔드 초보자이므로 **학습 곡선이 낮고 자료가 풍부한 메인스트림 스택**을 채택.

## 사이트 구조

### 메인 페이지 `/` (단일 페이지 스크롤)
```
Hero (이름 + 역할 한 줄)
   ↓
About / 자기소개서
   ↓
Resume / 이력서·경력기술서 (학력·경력·기술스택·자격증)
   ↓
Career Roadmap (세로 타임라인, 최신순)
   ┌─────────────────────────┐
   │ ⊝  Coming soon...       │   ← 고정 플레이스홀더 (블러)
   │    준비 중               │
   └─────────────────────────┘
        │
   2024 ●─ 게임 프로젝트 (취업 후)        [compact]
        │
   2023 ●─ 비동기 IOCP 네트워크 라이브러리 [compact]
        │
   2021 ●─ AI 학부모안심서비스            [compact]
        │
   2020 ●─ 라인트레이서                  [compact]
   ↓
Contact (이메일·GitHub)
```

- **최상단 플레이스홀더**: 항상 자리하는 단일 블러 카드. 클릭 불가. "지속 업데이트" 시그널.
- **compact 항목**: 구조도/상세 없는 4개 (현재 완료된 라인트레이서·AI 안심·IOCP·게임). 작은 점 + 한 줄 설명 + 기술 태그. 클릭 불가.
- **새로 완료된 프로젝트가 추가되면**: 일반 카드형 항목으로 렌더되고 클릭 시 상세 페이지로 이동. 자연스럽게 플레이스홀더와 compact 항목들 사이에 연도순으로 배치됨.

### 프로젝트 상세 `/projects/[slug]`
- **완료된 프로젝트 중 구조도·상세를 가진 것만** 라우트 존재
- v1 출시 시점에는 라우트가 0개 (compact 4개는 상세 없음). 향후 새 프로젝트 추가 시 자동 생성됨
- 헤더: 제목 + 연도 + 기술 스택 칩
- 본문: 구조도 이미지 + 개요/주요 기능/회고 (MDX로 자유 작성)

### 타임라인 정렬 규칙
1. **항상 최상단**: "Coming soon" 플레이스홀더 카드 1개 (고정)
2. 그 아래: 모든 프로젝트 entry를 `year` 내림차순 + 같은 해 내에서는 `order` 내림차순으로 정렬
3. compact 항목과 MDX 항목이 한 리스트에서 통합 정렬됨

## 디자인 시스템

### 컬러 팔레트 (`globals.css` CSS 변수)
```css
--bg:     #FAFAF9;   /* 살짝 따뜻한 흰색 */
--fg:     #18181B;   /* 거의 검정 */
--muted:  #71717A;   /* 회색 (보조 텍스트) */
--rule:   #E4E4E7;   /* 헤어라인, 타임라인 레일 */
--accent: #B45309;   /* 단일 액센트 (앰버 톤) */
--locked: #A1A1AA;   /* 플레이스홀더 등 흐린 톤 */
```
액센트는 **최소한으로만** 사용 (최신 프로젝트 점, 호버 밑줄, 배지 보더 정도).

### 타이포그래피
- 본문: Pretendard, `text-base md:text-lg`, `leading-relaxed`
- Hero: `text-5xl md:text-7xl tracking-tight`
- 섹션 제목: `text-3xl md:text-4xl tracking-tight`
- 날짜/코드/캡스: JetBrains Mono, `tracking-wide`
- 본문 폭: `max-w-3xl`, 타임라인 폭: `max-w-5xl`

### 타임라인 레이아웃
- CSS Grid 2-column: `[연도 80~120px] [본문 1fr]`
- 좌측 세로선(`::before` 1px) + 각 항목 위치에 12px 원형 점
- compact 항목: 점 8px, 카드 배경 없이 한 줄 설명
- 일반 항목 (향후 추가될 완료 프로젝트): 카드형 + 커버 썸네일
- 플레이스홀더 카드: 점선 테두리 + 블러 + "준비 중" 라벨, 점은 빈 원
- 모바일 (`< md`): 단일 컬럼, 연도가 제목 위에 작게

## 개발·배포 워크플로 (사용자가 헷갈리지 않도록)

**핵심**: localhost에서 개발하고, 만족스러울 때만 GitHub에 push. push가 일어나면 Vercel이 자동으로 빌드해서 라이브 사이트에 반영.

| 단계 | 비유 (C++) | 웹 (Next.js + Vercel) |
|---|---|---|
| 작업 중 | 디버그 빌드 + 실행 | `npm run dev` → `localhost:3000` 에서 실시간 확인 |
| 코드 저장 | 컴파일 | 파일 저장하면 브라우저 자동 새로고침 (HMR) |
| 출시 | 릴리즈 빌드 + 배포 | `git push` → Vercel이 90초 내 자동 빌드·배포 |

기본 흐름:
```
[로컬 코드 수정] → [localhost:3000 확인] → [git commit & push]
                                                    ↓
                                            [Vercel 자동 빌드]
                                                    ↓
                                          [라이브 사이트 갱신]
```

**Pull Request 사용 시 더 안전 (선택)**:
- 새 브랜치에서 작업 → PR 생성 → Vercel이 **미리보기 URL** 자동 생성
- 미리보기에서 확인 후 main에 머지 → 그제서야 production 반영
- v1 단계에서는 main에 직접 push 해도 무방

## 새 프로젝트 추가 방법 (사용자 워크플로)

1. `src/content/projects/<slug>.mdx` 파일 생성 (frontmatter + 본문 작성). `status: "completed"`, 적절한 `year`, `order` 지정.
2. `public/images/projects/<slug>/` 에 cover/diagram 이미지 추가.
3. 로컬에서 `npm run dev` 로 확인 → 잘 보이면 `git add . && git commit -m "Add project: <slug>" && git push`.
4. Vercel이 약 90초 내에 자동 배포. 라이브 사이트의 타임라인에 새 항목 자동 등장 + `/projects/<slug>` 페이지 자동 생성.

진행 중 프로젝트 표시: `status: "in-progress"` 로 두면 "진행 중" 배지가 붙음. 완료되면 `"completed"` 로 변경.
